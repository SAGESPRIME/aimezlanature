# Playbook — pratiques qui marchent

[2026-07-20] | Recherche SEO DataForSEO à petit budget : 1 appel search_volume (35 seeds) + keyword_suggestions + related_keywords + 5 SERP live + ranked_keywords du leader = 0,17 $ | Vision complète marché + concurrents pour moins de 20 centimes
[2026-07-20] | Pattern d'appel API : payload JSON dans un fichier scratchpad + curl -u "$(cat creds.txt)" + petit node -e pour extraire | Traçable (JSON bruts conservés), rejouable, identifiants jamais dans le projet
[2026-07-20] | Analyser les ranked_keywords du concurrent n°1 (lesvertsmoutons.com) plutôt que deviner | A révélé le modèle gagnant : 1 fiche produit forte + 4 guides blog qui verrouillent les requêtes confiance/usage
[2026-07-20] | Pages guide Astro : contenu (FAQ, tableaux, étapes) en tableaux d'objets dans le frontmatter + composants partagés Faq/CtaPacks + helpers lib/seo.ts | Fichiers < 300 lignes, schema JSON-LD généré depuis la même source que l'affichage (zéro divergence), vérifiable par grep sur dist/
[2026-07-20] | Avant de rédiger des mentions légales/CGV, vérifier via WebFetch si un site déjà en ligne pour la marque existe (ex. aimezlanature.fr) plutôt que d'inventer ou de mettre un placeholder | Récupère les vraies infos légales (raison sociale, SIRET, hébergeur) sans halluciner ni bloquer sur une question à l'utilisateur
[2026-07-20] | Récupérer les assets (photos produit, logo) depuis le site en ligne : `curl` du HTML brut puis grep sur `data-large_image`/`wp-post-image` — WebFetch ne voit que les placeholders lazy-load | Vraies photos du client, aucune image inventée ; le sitemap produit du site en ligne révèle aussi le catalogue réel (a permis de détecter que le Pack 35 n'existait plus)
[2026-07-20] | Rendre optionnels les champs d'asset dans les schemas (`buildProductSchema(p, imageUrl?)`) + une map slug→image dans un seul module | Un produit sans photo dégrade proprement (visuel de substitution, champ omis) au lieu d'émettre une URL 404 ; ajouter une photo = une ligne à changer
[2026-07-20] | Sur un site statique avec l'adaptateur Cloudflare, forcer `cloudflare({ imageService: 'compile' })` | Les WebP sont générés au build par sharp et servis en statique avec cache immuable, au lieu de passer par Cloudflare Images (facturé) à chaque affichage
[2026-07-20] | Avant de reprendre une fiche produit du site existant, vérifier que la note affichée appartient bien au produit et pas à un bloc de cross-sell (ici 4,94 et 4,91 étaient les packs suggérés, la gourde n'avait aucun avis) | Évite de publier une note inventée — interdit par Google et trompeur ; rendre `rating`/`reviewCount` optionnels résout le cas proprement
[2026-07-20] | Ne pas recopier les allégations marketing du site existant (« 2,3x mieux qu'un filtre », « dure 10 ans ») sans source | Le site assume une ligne honnête (page À propos, guide Avis) : une allégation non sourcée la contredit et expose côté allégations commerciales
[2026-07-20] | Récupérer les VRAIS avis clients depuis le JSON-LD WooCommerce du site en ligne (curl + regex sur "reviewBody"/"name") | Preuve sociale authentique et défendable (noms + textes exacts), zéro invention
[2026-07-20] | Contraste : calculer les ratios WCAG en node (petite fonction luminance) pour choisir une teinte de remplacement AVANT d'éditer, puis sed sur toutes les occurrences | Une seule passe, 44 occurrences corrigées d'un coup, Lighthouse A11y 95→100
[2026-07-20] | CSP stricte sur site statique Astro/Cloudflare : `inlineStylesheets: 'never'` + JS dans public/js/*.js + polices @fontsource auto-hébergées → `default-src 'self'` sans unsafe-inline (les JSON-LD inline ne sont pas concernés par la CSP) | Sécurité maximale sans rien casser, vérifiable en grepant dist pour <style et <script inline
[2026-07-20] | Animations légères sans lib : [data-reveal] + IntersectionObserver + CSS sous `@media (scripting: enabled) and (prefers-reduced-motion: no-preference)` | Contenu visible sans JS, respect reduced-motion, CLS 0 mesuré, zéro dépendance
[2026-07-20] | Section blog : articles neufs sur les requêtes NON couvertes par les piliers (trous du mapping DataForSEO) plutôt que recréer les anciens posts déjà 301-és | Zéro cannibalisation ; vérifiable en contrôlant l'unicité des questions FAQPage sur tout le dist (script node, 70 questions, 0 doublon)
[2026-07-23] | Vérifier une feature JS interactive quand l'onglet d'automatisation est instable : `requestAnimationFrame` et IntersectionObserver ne se déclenchent pas dans un onglet en arrière-plan (non peint), et un `scrollTo` programmatique affiche une capture blanche. Contourner en (1) lisant l'état du DOM directement (`classList`, `.hidden`, `localStorage`) qui, lui, est fiable ; (2) utilisant le VRAI scroll via le tool `computer` (roulette) + capture, qui force le rendu | Preuve terrain d'un comportement au défilement/clic sans se fier à des API tributaires du paint. A permis de confirmer barre collante + ajout panier + synchro prix
[2026-07-23] | Barre d'achat collante sans lib ni dépendance de prix dupliquée : bouton `data-ajouter-panier` intercepté par délégation dans cart.js (même chemin que le BuyBox), total recopié depuis `[data-palier-total]` du palier coché | Zéro divergence de prix (33,84 € identique barre/panier vérifié), une seule source de vérité pour le calcul
[2026-07-23] | Skills design ui-ux-pro-max (recherche best-practices, base locale) puis impeccable (critique/audit) : traiter la sortie de la base comme INSPIRATION, pas comme vérité — elle a proposé une DA bleu « Liquid Glass » qui aurait cassé la marque naturelle existante. Garder le pattern transposable (Before-After), jeter le reste | Recommandations fondées + marque préservée ; le pattern retenu vient des données, la DA reste celle du client
[2026-07-23] | Nouveau bloc de vente sur fond coloré : mesurer le contraste WCAG de CHAQUE texte au navigateur via un resolver canvas (gère oklch/oklab/color() de Tailwind v4 que la regex ne lit pas) AVANT de considérer le travail fini | Le texte « avant » gris-sauge à 70 % passait de justesse (5,30:1) — sans mesure j'aurais pu shipper un échec a11y qui contredit la rigueur des audits précédents
[2026-07-23] | Avant de construire une page « métier » (offre revendeur B2B), vérifier la demande réelle avec DataForSEO AVANT d'écrire quoi que ce soit : « grossiste perles de céramique », « perles de céramique en gros », « revendeur produits écologiques » = 0 recherche/mois en France | A évité de bâtir un cluster SEO inutile. La page a été conçue comme un support de démarchage (une seule page, lien discret en pied de page, hors menu principal) au lieu d'une machine à trafic — pour 0,05 $ de requête
[2026-07-23] | Formulaire public sur site à CSP stricte : `form-action 'self'` interdit Tally/Google Forms/Formspree, et un captcha tiers (Turnstile) obligerait à relâcher `script-src 'self'`. Solution : `<form method="post">` natif vers une route Astro `prerender = false` sur le Worker, anti-robot par champ piège + horodatage, redirection 303 vers une page de remerciement | Formulaire fonctionnel SANS JavaScript (prouvé : POST curl → 303 vers /revendeurs/merci/), zéro script tiers, CSP intacte. Le JS ne fait qu'améliorer (les champs saisis survivent à une erreur)
[2026-07-23] | Données non validées par le client : structurer la page pour l'absence de donnée plutôt que d'attendre. `paliersRemise: PalierRemise[] = []` dans le fichier de données + un ternaire dans la page (tableau si rempli, texte de principe si vide) | La page est livrable aujourd'hui sans inventer un seul chiffre ; le jour où le marchand valide sa grille, il suffit de remplir le tableau — aucun fichier de présentation à retoucher. Vérifié par un script qui compte les pourcentages dans dist/
[2026-07-23] | Reprendre les IDÉES d'une présentation/maquette externe sans en copier le CONTENANT : d'une présentation interactive à défilement (incompatible CSP + mauvais pour le SEO car contenu JS-dépendant), extrait 3 patterns transposables — rituel en 3 verbes, « nommer le problème avant la solution », e-book en argument d'achat — portés à la main en .astro statique. Chaque idée passée au crible « demande réelle ? » (DataForSEO : 0/mois → conversion, pas SEO) et « déjà sur le site ? » (grep avant d'écrire) | Le site gagne la valeur narrative de la présentation sans sa dette technique ; zéro doublon (le bloc avant/après existant n'a pas été redondé), zéro chiffre inventé. Vérifié au navigateur (contrastes 7-17:1, 0 débordement 390px, PDF servi en application/pdf)
